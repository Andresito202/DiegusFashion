1. DESCRIPCION DEL PROYECTO
================================================================

Diegu's Fashion es una tienda online de chaquetas por mayor y
detal ubicada en el C.C. Gran San, San Victorino, Bogota D.C.

Pagina web tipo catalogo con panel de administracion, pedidos
por WhatsApp y pagos por Nequi, cuenta bancaria o contra entrega.

Toda la informacion (productos, categorias, config) se almacena
en Firebase Firestore y se actualiza en TIEMPO REAL.

3. TECNOLOGIAS UTILIZADAS
================================================================

FRONTEND:
- HTML5 semantico con meta tags SEO (OG, Twitter Card, Schema.org)
- CSS3 con variables CSS y diseno responsive (mobile first)
- JavaScript vanilla (sin frameworks)
- Bootstrap 5.3 (grid y utilidades)
- Font Awesome 6.4 (iconos)
- Google Fonts - Poppins

BACKEND / BASE DE DATOS:
- Firebase Firestore (base de datos en la nube, tiempo real)
- Firebase Authentication (login del admin con email/password)
- Firebase CDN compat v10.7.0

SERVIDOR:
- .htaccess para Apache/cPanel
- Compresion GZIP
- Cache de navegador (CSS/JS 1 mes, imagenes 6 meses)
- Headers de seguridad (HSTS, X-Frame-Options, nosniff, etc.)

IMAGENES:
- Archivos locales en carpeta img/
- Soporte para URLs externas (ImgBB u otro CDN)
- Hasta 4 imagenes por producto

7. SEGURIDAD IMPLEMENTADA
================================================================

FIREBASE:
- Reglas Firestore restringidas a un UID admin especifico
- Solo el admin puede escribir (productos, categorias, config)
- Lectura publica para que la tienda funcione
- Authentication con email/password (un solo usuario admin)

SERVIDOR (.htaccess):
- Bloqueo de archivos sensibles (.env, .gitignore, .md, .txt, .json)
- Headers: X-Content-Type-Options, X-Frame-Options, X-XSS-Protection
- Referrer-Policy strict-origin-when-cross-origin
- Permissions-Policy (bloquea camara, microfono, geolocalizacion)
- Strict-Transport-Security (HSTS 1 ano)
- Proteccion de firebase-config.js contra acceso externo directo

CODIGO:
- CONFIG dentro de IIFE (no accesible desde consola)
- Sanitizacion de datos en admin antes de guardar
- escapeHtml en toda renderizacion dinamica (previene XSS)
- admin.html con meta robots noindex, nofollow
